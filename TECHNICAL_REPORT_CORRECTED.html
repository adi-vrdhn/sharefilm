1. Abstract

FilmShare is a multifaceted, full-stack social media application designed to revolutionize how users discover, share, and consume movies. The platform leverages cutting-edge web and mobile technologies to provide seamless cross-platform experiences across iOS, Android, and web browsers. This comprehensive technical report documents the architecture, implementation, and innovative features of FilmShare, including an intelligent recommendation engine, real-time social features, and advanced taste profiling algorithms. The system integrates The Movie Database (TMDB) API for content aggregation, implements robust security mechanisms including JWT authentication and OAuth 2.0, and employs PostgreSQL for persistent data storage. With over 14 distinct functional modules and real-time socket-based communication, FilmShare serves as a comprehensive case study in modern full-stack application development. This report details the system's architectural design, implementation strategies, database schema, security infrastructure, and deployment methodology.

Keywords: Recommendation Engine, Full-Stack Development, React, React Native, Real-Time Communication, Movie Social Network, Machine Learning, Web Architecture, Mobile Development, Database Design

2. Introduction

2.1 Background and Motivation

The exponential growth of Over-The-Top (OTT) streaming platforms has led to what research terms the "paradox of choice" phenomenon. While users have access to millions of movies across multiple streaming platforms, the overwhelming selection has created significant friction in content discovery. Traditional recommendation algorithms often rely solely on viewing history and genre preferences, failing to capture nuanced taste preferences and social dynamics that influence viewing decisions.

FilmShare addresses this challenge by creating a social ecosystem where movie recommendations flow through trusted friend networks, combined with intelligent algorithmic matching based on comprehensive taste profiling. This hybrid approach merges collaborative filtering with content-based recommendation strategies, enriched by social signals.

2.2 Problem Statement

Current streaming platforms suffer from several critical limitations:

  • Isolated Recommendations: Algorithms fail to incorporate social context and friend preferences
  • Discovery Friction: Users spend significant time browsing without finding satisfactory content
  • Platform Fragmentation: Content scattered across multiple streaming services without unified discovery
  • Limited Social Integration: Minimal mechanisms for sharing and discussing movie preferences
  • Inadequate Taste Profiling: Generic recommendations not tailored to individual aesthetic preferences
  • Engagement Metrics: Limited mechanisms to track and visualize personal viewing patterns

2.3 Proposed Solution

FilmShare proposes an integrated solution architecture comprising:

  • Multi-Platform Accessibility: Web, iOS, and Android applications with synchronized data
  • Intelligent Recommendation Engine: Hybrid recommendation system combining collaborative and content-based filtering
  • Taste Profile System: Comprehensive user profiling based on movie attributes, genres, and ratings
  • Social Network Features: Friend connections, movie sharing, real-time notifications
  • Advanced Analytics: Dashboard visualizations for personal viewing patterns and statistics
  • Real-Time Communication: Socket-based messaging and notifications

2.4 Scope and Objectives

Primary Objectives:

  1. Develop a scalable, secure multi-platform application architecture
  2. Implement an intelligent recommendation engine with taste profiling
  3. Create real-time social features with WebSocket communication
  4. Ensure robust data security and user privacy
  5. Provide comprehensive analytics and user engagement metrics

4. Database Design

4.1 Schema Overview

The system employs a relational PostgreSQL schema with 11 primary entities:

4.2 Core Entities

User Entity: Stores user information including email, name, password hash, OAuth identifiers, avatar, bio, and account status. Includes fields for failed login attempts and account lockout tracking.

Movie Entity: Stores movie information from TMDB including title, description, poster URL, release date, rating, vote count, and genre arrays.

UserMovie Entity: Tracks movies sent between users with status (pending/watched/ignored), timestamps for when sent and watched.

Rating Entity: Stores user ratings and reviews for movies on 1-10 scale with review text.

Friendship Entity: Manages friend connections with status tracking (pending/accepted/blocked).

Message Entity: Stores direct messages between users with read status indicators.

Notification Entity: Tracks notifications (friend requests, movie sends, etc.) with read status.

Taste Profile Entities:

  • user_taste_profiles: Stores favorite genres, preferred rating scores
  • user_taste_vectors: 14-dimensional genre vectors, embeddings for actors/directors, thematic preferences

Party & Analytics Entities: Supports movie parties, shared viewing events, and user statistics.

4.3 Indexing and Integrity

Key Indexes:

  • idx_users_email: Email lookups
  • idx_user_movies_user_id, idx_user_movies_status: Movie tracking queries
  • idx_ratings_user_movie: User rating queries
  • idx_messages_sender_recipient: Message retrieval
  • idx_notifications_user_id: Notification queries
  • idx_movies_tmdb_id: Movie lookups

Data Integrity Constraints:

  • Foreign Key Constraints: All relationships enforced with CASCADE/RESTRICT policies
  • Unique Constraints: Email (users), TMDB ID (movies), composite keys for relationships
  • Check Constraints: Rating values (1-10), friendship status validation
  • NOT NULL Constraints: Critical fields protected from null values

7. Core Features Implementation

7.1 Intelligent Recommendation Engine

Algorithm Architecture - Four-Tier Recommendation Stack:

Layer 1: Collaborative Filtering
  Input: User-item ratings matrix
  Algorithm: Cosine similarity
  Output: User-to-user similarity scores
  
Layer 2: Content-Based Filtering
  Input: Movie attributes (genre, director, actors, themes)
  Algorithm: TF-IDF vectorization
  Output: Movie-to-movie similarity
  
Layer 3: Taste Vector Matching
  Input: User taste profiles (14-dim vectors)
  Algorithm: Euclidean distance in feature space
  Output: Personalized recommendation weights
  
Layer 4: Social Signals
  Input: Friend ratings, friend watchlists
  Algorithm: Weighted friend influence
  Output: Social-based scoring

Final Recommendation Score:

Score = (0.4 × Collaborative) + (0.3 × Content-Based) + 
         (0.2 × Taste Vector) + (0.1 × Social Signal)

Taste Vector Generation:

  • 14-dimensional genre weight vector (normalized to [0,1])
  • Director and actor embeddings (TF-IDF derived)
  • Thematic preference encoding
  • Real-time weight adjustments based on user feedback
  • Updated with: new_vector = 0.7 × old + 0.3 × interaction

7.2 Movie Matching System

Purpose: Find compatible movies for pairs of friends with shared taste preferences.

Algorithm:

  • Get top 100 movies for each user based on preferences
  • Find intersection of common candidates
  • If common movies exist, rank by average compatibility score
  • If no intersection, create synthetic matches using average preference vector
  • Return 30 top-ranked compatible movies

Compatibility Score:

taste_distance = euclidean_distance(user_a_vector, user_b_vector)
compatibility_score = 1 - normalized_distance

Match Levels:
- score > 0.85: Perfect Match
- score > 0.70: Excellent Match  
- score > 0.55: Good Match
- score > 0.40: Fair Match
- score ≤ 0.40: Different Tastes

8. Security Implementation

8.1 Authentication Security

Password Storage:

  • Algorithm: bcryptjs with 12 salt rounds
  • Never stores plaintext passwords
  • Verification: bcryptjs.compare(input, stored_hash)

JWT Token Security:

  • Algorithm: HS256 (HMAC-SHA256)
  • Expires: 24 hours
  • Storage: HTTP-only cookie (web), Secure storage (mobile)
  • Includes: user_id, email, issued_at, expiration, roles

OAuth 2.0 Integration:

  • Google OAuth for federated authentication
  • No password storage for OAuth users
  • Automatic user creation on first login
  • Token refresh and validation mechanisms

8.2 Authorization and Access Control

Route Protection:

  • Public Routes: Sign up, login, movie search, home
  • Protected Routes: Profile, friends, messages, watchlist, recommendations
  • Admin Routes: User management, analytics export (future)

Resource-Level Authorization:

  • Users can only access their own data
  • Friendship required to view certain profiles
  • Message access restricted to sender/recipient
  • Enumeration attacks prevented via consistent error messages

8.3 Input Validation and Sanitization

Express-Validator Integration:

  • Email validation with normalization
  • Password strength requirements: 12+ chars, uppercase, lowercase, number, special char
  • Name validation: 2-100 characters, trimmed
  • All inputs validated on server-side

Input Sanitization:

  • HTML/script tag removal
  • XSS attack prevention
  • CSRF token validation
  • SQL injection prevention via parameterized queries

8.4 CORS and SSL/TLS Security

CORS Configuration:

  • Whitelist allowed origins
  • Allowed methods: GET, POST, PUT, DELETE
  • Credentials: true for authenticated requests
  • Max age: 86,400 seconds (24 hours)

SSL/TLS:

  • HTTPS enforced in production
  • Helmet.js security headers (HSTS, CSP, X-Frame-Options)
  • Secure cookies (httpOnly, Secure flags)
  • SameSite cookie attributes

8.5 Account Security Features

Account Lockout Mechanism:

  • 5 failed login attempts trigger 15-minute lockout
  • Lockout countdown displayed to user
  • Audit logging of failed attempts
  • IP-based rate limiting for brute force protection

Audit Logging:

  • Records: login attempts, failed authentication, privilege changes
  • Tracks: user ID, IP address, timestamp, action
  • Enables detection of suspicious behavior
  • Supports incident investigation and forensics

9.2 Caching Strategy

Recommended Redis Caching Layers:

  • User recommendations (TTL: 24 hours)
  • Movie details (TTL: 7 days)
  • Trending movies (TTL: 1 hour)
  • User taste profiles (TTL: 48 hours)
  • Friend lists (TTL: 6 hours)

Cache Invalidation Strategy:

  • User rates movie → Invalidate user's recommendations
  • New movie sent → Invalidate user's feed
  • Profile updated → Invalidate taste profile cache
  • Friendship change → Invalidate friend list cache
